
# Second-Semester-Java-OOP-Encapsulation-Systems
A dual-module Object-Oriented Java system implementing structured data validation, strict access control encapsulation, and deep-copy mechanics for healthcare records and temporal scheduling models.

# Secure Data Management & Scheduling Systems

Welcome to my Java development repository! This project implements structured, encapsulated analytical backends built during my 2nd semester of Software Engineering. It provides reliable class models to process medical registry files and calculate variable scheduling intervals under strict object-state data validations.

## Key Topics Covered

* **Data Architecture Encapsulation:** Locking mutable attributes via private boundaries and routing visibility safely through verified getter and setter accessors.
* **Factory Defenses:** Engineering static creation checkpoints that block empty references, illegal inputs, and out-of-bounds parameters before memory instantiation.
* **Deep Reference Separation:** Building explicit copy constructors that clone target object state matrices into completely isolated operational memory tracks.

## Featured Core Modules

This repository centers on two analytical engineering modules:

* **PatientRecord.java** - Manages unique clinic profiles, verifies matching categorical blood lines, monitors visitation tracks, and combines matching patient histories.
* **TimeInterval.java** - Parses hours, minutes, and seconds data layers, executes runtime scale shifts, and auto-updates operational complexity tags based on dynamic structural lengths.

## How to Run These Programs

1. Open your terminal environment inside the project directory.
2. Compile the target module files:
   ```bash
   javac PatientRecordTest.java TimeIntervalTest.java
   ```
3. Execute the compiled application drivers:
   ```bash
   java PatientRecordTest
   java TimeIntervalTest
   ```
